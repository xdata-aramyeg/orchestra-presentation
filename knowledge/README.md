# Knowledge Base

The team's shared, continuously-improved knowledge. **Owned by the PM**, read by everyone.

- **`conventions.md`** — the stack + coding standards every coder follows.
- **`workflow-discipline.md`** — the hard coordination rules (barriers, idle-during-QA, gates).
- **`course-corrections/`** — a running log of steering and lessons learned, so the team
  self-corrects and never repeats a mistake.
- **`presenter-cues-ru.md`** — printable cue cards for on-screen presenter mode (mirror of
  `orchestra-site/content/presenter-cues.ts`); one cue per demo-day beat (SAY · DROP · action).

## How the team uses this

- **Every teammate** reads `conventions.md` and `workflow-discipline.md` before acting, and
  skims `course-corrections/` for anything relevant to its role.
- **The Lead** re-reads `course-corrections/` before each phase/barrier and applies every entry.
- **The PM** keeps all of this current. When the human course-corrects, or the team hits a
  recurring snag, the PM **adds a course-correction entry** (from `course-corrections/TEMPLATE.md`)
  so the lesson persists. This is the team's continuous-improvement loop.

The agent-teams feature itself is documented in `../docs/agent-teams.md`.
