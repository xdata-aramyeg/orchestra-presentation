# Course Corrections

A running log of steering and lessons learned, so the team self-corrects and never repeats
a mistake. **Owned by the PM** (continuous improvement); **read by the Lead** before each
phase and barrier, and applied.

## How it works
- When the human course-corrects the team, or the team hits a recurring snag, the PM adds a
  new numbered entry here using `TEMPLATE.md`.
- The Lead reads every entry before each phase/barrier and applies it.
- Entries are append-only history — supersede with a new entry rather than rewriting old ones.

## Index
- `0001-idle-during-qa.md` — devs stay idle while QA tests.
- `0002-lead-never-writes-code.md` — the Lead orchestrates, never hand-codes.
- `0003-qa-tests-only-via-chrome-mcp.md` — QA tests via Chrome MCP + computer use; writes only test cases.
